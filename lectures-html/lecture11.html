Cluster Analysis

What changes in an unsupervised problem?

  • Classification starts with known labels. Clustering starts with structure and asks whether groups exist at all.
  • We need a notion of similarity before we can talk about a cluster.
  • Different clustering algorithms make different promises.
    • Hierarchical clustering gives a nested tree.
    • K-means gives compact partitions.
    • K-medoids swaps means for representative observations.
    • DBSCAN keeps dense shapes and leaves sparse points as noise.

Lecture roadmap

  1. Build dissimilarities for multivariate data.
  2. Cut a dendrogram and check whether the resulting groups are stable.
  3. Compare centroid and medoid based partitioning.
  4. Use density clustering when convex clusters fail.
  5. Finish with a modern workflow: parquet, HDBSCAN, UMAP, and the Qatar Cars exercise.

Dissimilarity matrices

  • Euclidean distance: \[ d_{st} = \sqrt{\sum_{j=1}^p (x_{sj} - x_{tj})^2} \]
  • Manhattan distance: \[ d_{st} = \sum_{j=1}^p |x_{sj} - x_{tj}| \]
  • Gower dissimilarity mixes numeric and categorical variables by rescaling each numeric field to its range and then combining numeric and factor mismatches.

Hierarchical Clustering

Agglomerative hierarchical clustering

  1. Start with each observation in its own cluster.
  2. Find the closest pair of clusters.
  3. Merge them.
  4. Repeat until only one cluster remains.

The only extra decision is how to define the distance between clusters.

  • Single linkage uses the closest pair.
  • Complete linkage uses the furthest pair.
  • Average linkage uses the mean pairwise distance.

Bootstrap stability: do the same clusters return?

  • Resample observations.
  • Refit the clustering on each resample.
  • Compare the resulting partition with the original.
  • A stable cluster should survive small perturbations to the data.

Bootstrap stability on monsters

The adjusted Rand index compares every pair of monsters across two partitions: did that pair stay together, or stay apart, in both clusterings? It then subtracts the agreement expected by chance, \[ \mathrm{ARI} = \frac{\sum_{ij}\binom{n_{ij}}{2} - E}{\frac{1}{2}\left[\sum_i \binom{a_i}{2} + \sum_j \binom{b_j}{2}\right] - E}, \] so 1 means a perfect match to the baseline clustering, 0 means about as much agreement as random assignment, and negative values mean worse than random agreement.

K-means and K-medoids

K-means minimises within-cluster variation

For Euclidean distance the objective is \[ \sum_{k=1}^K \sum_{C(i)=k} \lVert x_i - \mu_k \rVert^2. \]

Algorithm:

  1. Initialise centroids.
  2. Assign each point to the nearest centroid.
  3. Recompute centroids.
  4. Repeat until the assignments stop changing.

K-medoids / PAM

  • Replace the centroid with a real observation, the medoid.
  • This makes the method less sensitive to outliers.
  • It also lets us use any dissimilarity, not just Euclidean distance.

When K-means and PAM disagree

  • K-means is faster and works well for compact numeric clusters.
  • PAM is more robust when outliers are likely or when the cluster center needs to be interpretable.
  • Both still force every observation into one of K groups.

Density Clustering

DBSCAN definitions

  • eps: neighborhood radius.
  • MinPts: minimum neighborhood size.
  • Core point: at least MinPts observations in its eps-neighborhood.
  • Border point: not core, but within the neighborhood of a core point.
  • Noise point: neither core nor border.

Why DBSCAN is different

  • It does not require K.
  • It can recover non-convex shapes.
  • It can leave points unclustered.
  • It is sensitive to scale and parameter choice.

Parameter choice: k-distance idea

  • Here MinPts = 5, so a point becomes core when it has 4 other points inside its eps neighborhood.
  • That makes the distance to the 4th nearest other point the smallest eps that would make that observation core.
  • Sort those distances and look for an elbow: dense-cluster points stay low, while border/noise points rise sharply.
  • Choosing eps near that bend usually keeps dense regions together without flooding sparse noise.
min_pts <- 5
k <- min_pts - 1  # 4 other neighbors, because the point itself is counted in MinPts
dmat <- as.matrix(dist(dbscan_data |> select(x, y)))
knn_distance <- apply(dmat, 1, function(row) sort(row)[k + 1])  # +1 because the self-distance is 0
tibble(rank = seq_along(knn_distance), kth_distance = sort(knn_distance)) |>
  ggplot(aes(x = rank, y = kth_distance)) +
  geom_line(colour = "#197278", size = 1.1) +
  labs(
    x = "Ordered points",
    y = paste0("Distance to ", k, "th nearest other point (MinPts = ", min_pts, ")")
  )

DBSCAN repeats: inspect an unvisited point, test whether it is core, expand through its neighbors if it is core, and otherwise leave it as provisional noise until a later core point reaches it.

HDBSCAN: the 2026 default when density varies

  • HDBSCAN extends DBSCAN into a hierarchy and avoids fixing a single eps.
  • It is a better default when dense and sparse clusters coexist.
  • In R, the standard entry point is dbscan::hdbscan().
library(dbscan)
hdbscan(monster_scaled |> select(all_of(numeric_cols)), minPts = 8)

UMAP in one slide

  • UMAP starts from local neighbor relationships in the original feature space and builds weighted affinities w_{ij}.
  • It then finds low-dimensional coordinates y_i whose low-dimensional affinities v_{ij} preserve those neighborhoods as well as possible.

\[ v_{ij} = \left(1 + a\lVert y_i - y_j \rVert^{2b}\right)^{-1} \]

\[ \mathcal{L} = \sum_{i \ne j} \left[ w_{ij}\log\frac{w_{ij}}{v_{ij}} + (1-w_{ij})\log\frac{1-w_{ij}}{1-v_{ij}} \right] \]

  • So UMAP 1 and UMAP 2 are not original variables. They are coordinates chosen to keep nearby monsters nearby in the map.

Suggested workflow for the Qatar Cars lab

  1. Start with summary() and a few scatterplots.
  2. Scale the numeric features before using Euclidean distance.
  3. Compare complete-linkage hierarchical clustering with K-means.
  4. Use PAM silhouettes to decide whether your chosen K is defensible.
  5. Try DBSCAN last on a small feature subset where the geometry is visible.

Summary

Cluster analysis summary

  • Distances create the problem definition, so scaling and variable choice come first.
  • Hierarchical clustering is good for structure discovery and explanation.
  • K-means and PAM are explicit partitioning tools once a candidate K is on the table.
  • DBSCAN is the right move when you care about shape or noise.
  • Modern practice adds parquet for exchange, HDBSCAN for density variation, and UMAP for visualization.
